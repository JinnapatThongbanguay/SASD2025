## HW3: วิเคราะห์ C# Class 3 ประเภท จาก .NET Based APIs
**แหล่งข้อมูล:** [Microsoft Learn - .NET 10.0 API ](https://learn.microsoft.com/en-us/dotnet/api/?view=net-10.0)

### 1. Utility Class
**Class:** `System.Math` (ทำหน้าที่เป็นเครื่องมือคำนวณทางคณิตศาสตร์)

* **เหตุผลที่จัดอยู่ในกลุ่มนี้:**
    * เป็น `static class` ทำให้ไม่สามารถสร้าง Instance หรือ object (ไม่สามารถใช้ new ได้) ขึ้นมาได้
    * ไม่มีการเก็บค่าหรือสถานะใด ๆ ไว้ในตัวมันเอง มีหน้าที่รับค่าเข้าไป (Input) และส่งผลลัพธ์ออกมา (Output) เท่านั้น
    * ฟังก์ชันสำเร็จรูปที่สามารถเรียกใช้งานได้ผ่านชื่อคลาส เช่น `Math.Sqrt()`, `Math.Abs()`, `Math.Max()`, `Math.Round()`, `Math.PI`

### 2. Mutable Class (แก้ไขค่าได้)
**Class:** `System.Collections.Generic.List<T>`

* **เหตุผลที่จัดอยู่ในกลุ่มนี้:**
    * สามารถแก้ไข เปลี่ยนแปลง หรือลบข้อมูลภายใน Object เดิมได้ตลอดเวลาหลังจากสร้างขึ้นมา
    * มี Methods ที่ใช้จัดการข้อมูลโดยตรง เช่น `.Add()`, `.Remove()`, `.Clear()` , `Insert()` ซึ่งจะเปลี่ยนข้อมูลในหน่วยความจำ object เดิม
    * ขนาดของข้อมูลภายในสามารถเปลี่ยนแปลงได้ตามการใช้งานโดยไม่ต้องสร้าง object ใหม่มาทดแทนตัวเดิม


### 3. Immutable Class (แก้ไขค่าไม่ได้)
**Class:** `System.String`

* **เหตุผลที่จัดอยู่ในกลุ่มนี้:**
    * เมื่อจองพื้นที่ในหน่วยความจำเพื่อสร้าง String แล้ว ข้อมูลในพื้นที่นั้นจะถูกล็อคไว้ทำให้ไม่สามารถแก้ไขค่าตัวอักษรข้างในได้
    * ทุกครั้งที่สั่งแก้ไขข้อความ เช่น `.ToUpper()` หรือ `.Replace()` ระบบจะสร้าง object ใหม่ ส่วน object เก่าจะมีค่าเท่าเดิม
    * เนื่องจาก Immutable Class แก้ไขค่าไม่ได้ ทำให้ปลอดภัยเมื่อใช้งานร่วมกันหลาย ๆ ส่วนในโปรแกรม เพราะมั่นใจได้ว่าค่าจะไม่ถูกเปลี่ยนแปลงโดยไม่ตั้งใจ
      
* ตัวอย่าง:
```csharp
  string str1 = "hello";
  string str2 = str1.ToUpper(); // สร้าง object ใหม่
  // str1 ยังเป็น "hello" 
  // str2 เป็น "HELLO"
```
